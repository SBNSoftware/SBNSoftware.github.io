2 # File: geometry_icarus.fcl
3 # Purpose: Configuration of LArSoft geometry and geometry service for ICARUS.
4 #
5 #
6 # Suggested usage
7 # ----------------
8 #
9 # Unless there are special needs, the geometry configuration should be included
10 # via a configuration bundle.
11 # The default one is called `icarus_geometry_services`.
12 # To use it in a complete configuration:
13 #
14 # services: {
15 # # ...
16 # @table::icarus_geometry_services # from geometry_icarus.fcl
17 # # ...
18 # } # services
19 #
20 # To add it to a preexisting configuration, overwriting any existing geometry
21 # configuration:
22 #
23 # services: {
24 # @table::services
25 # @table::icarus_geometry_services # from geometry_icarus.fcl
26 # } # services
27 #
28 #
29 # Options
30 # --------
31 #
32 # Currently the following configurations are provided:
33 #
34 # * `icarus_geometry_services`: current default configuration; also:
35 # * `icarus_overburden_geometry_services`:
36 # default configuration, but with concrete overburden
37 # * `icarus_no_overburden_geometry_services`:
38 # default configuration, but without concrete overburden
39 # * geometry with 9 meter long wires for the first induction plane, on split
40 # TPCs: this naming is left for legacy purposes only, and it is now the
41 # default (as above)
42 # * `icarus_geometry_services_legacy_icarus_splitwires`: legacy geometry
43 # configuration for samples simulated with icaruscode earlier than v09_18_00,
44 # with no overburden; also related:
45 # * `icarus_geometry_services_overburden_legacy_icarus_splitwires`
46 # (added 3-m concrete overburden)
47 # * `icarus_geometry_services_no_overburden_legacy_icarus_splitwires`
48 # (with no 3-m concrete overburden: as the default)
49 #
50 #
51 # Since the documentation is often not updated, check the definitions below
52 # to see which are the current defaults.
53 #
54 #
55 # Notes
56 # ------
57 #
58 # A special policy in ICARUS is to have a copy of the channel mapping algorithm
59 # configuration in the `Geometry` service configuration.
60 # This is required by the off-art geometry loading utilities in ICARUS.
61 # One day LArSoft will also support something like that and that will become
62 # the only location of the channel mapping configuration.
63 #
64 #
65 # Changes
66 # --------
67 #
68 # 20200709 (petrillo@slac.stanford.edu)
69 # adopted split wire geometry fixed by
70 # Alessandro Menegolli (alessandro.menegolli@unipv.it)
71 # 20220505 (petrillo@slac.stanford.edu)
72 # geometry default changed to include overburden
73 #
